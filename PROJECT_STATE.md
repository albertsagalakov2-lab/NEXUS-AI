# PROJECT_STATE

Короткая памятка по текущему состоянию проекта `NEXUS-AI`.

## Стек

- Next.js 16 (App Router)
- React 19
- TypeScript
- Tailwind CSS v4
- Radix UI / shadcn-подобные компоненты
- Supabase (`@supabase/ssr`, `@supabase/supabase-js`)
- Lucide Icons

## Структура

- `app/` — основные страницы, layout и API routes
- `app/(dashboard)/` — защищённая рабочая зона: чат, изображения, видео, профиль
- `app/api/` — серверные ручки для чатов, сообщений, изображений, видео и appearance
- `components/` — общие UI-компоненты, sidebar, brand, mobile nav
- `components/ui/` — базовые UI-элементы
- `lib/` — утилиты и Supabase-клиент/сервер
- `hooks/` — пользовательские хуки
- `public/` — изображения и иконки

## Основные маршруты

- `/` — главная страница
- `/sign-in` — вход
- `/sign-up` — регистрация
- `/chat` — чат
- `/image` — генерация изображений
- `/video` — видео-раздел
- `/profile` — профиль
- `/profile/appearance` — настройки внешнего вида
- `/pricing` — тарифы

## Что уже реализовано

- Авторизация через Supabase
- Рабочий dashboard-layout с sidebar и mobile bottom nav
- Чат с историей, отправкой сообщений и отдельными состояниями загрузки
- `/image`-раздел с генерацией, историей и настройками модели/формата
- `/video` как отдельный раздел
- `/profile` и appearance-настройки
- API routes для чатов, сообщений, изображений и видео
- Локальные визуальные правки под тёмную тему и чистый рабочий интерфейс

## Известные проблемы

- Для входа нужен корректный `.env.local` в той же рабочей копии, где запущен сервер.
- Если сервер поднят из другой папки или с тестовыми env-переменными, Supabase-вход может не работать.
- В `globals.css` ещё есть довольно широкие глобальные стили и переопределения; их лучше точечно упрощать дальше.
- Некоторые текстовые строки в терминале могут выглядеть искажённо из-за кодировки, но в браузере это не всегда проблема.
